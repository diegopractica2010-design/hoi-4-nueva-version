 4.3  Decisiones politicas
----------------------------------------------------------------------
 - Movilizacion: reclutas +10, estabilidad -5
 - Embargo: relaciones -50, PP +10

 FASE 13 — DEPURACION TOTAL POR EL AGENTE
================================================================================
 Objetivo: El agente (yo, la IA) ejecuta una depuracion total del proyecto
 completo. No es una auditoria pasiva - es una busqueda activa de errores,
 deuda tecnica, code smells, fallos logicos y problemas de rendimiento.

----------------------------------------------------------------------
 13.1  Depuracion archivo por archivo
----------------------------------------------------------------------
 - Revisar CADA script .gd en el proyecto (uno por uno):
   * Errores de sintaxis que el compilador no atrapo
   * Variables sin usar o mal tipadas
   * Funciones muertas (nunca llamadas)
   * Logica incorrecta (comparaciones, condiciones, loops)
   * Posibles null pointer exceptions
   * Conexiones de seniales rotas
   * Recursos que no se liberan (memory leaks)
 - Checklist por archivo:
   [ ] Compila sin warnings
   [ ] No tiene codigo comentado (a menos que sea intencional)
   [ ] No tiene print() de debug olvidados
   [ ] No tiene TODO/FIXME/HACK (a menos que sea deuda aceptada)
   [ ] Todas las variables tienen tipo explicito
   [ ] Todas las funciones tienen tipo de retorno
   [ ] No hay cadenas magicas (usar constantes)
   [ ] Maneja casos null/empty

----------------------------------------------------------------------
 13.2  Depuracion capa a capa
----------------------------------------------------------------------
 - Por cada una de las 25 capas identificadas en COMPARATIVA_HOI4.md:
   * Revisar la implementacion completa de la capa
   * Verificar que las conexiones entre capas funcionan
   * Comprobar que los datos de entrada/salida son correctos
   * Identificar dependencias circulares
   * Buscar cuellos de botella de rendimiento
   * Verificar que los signals/slots estan conectados y no generan
     duplicados

----------------------------------------------------------------------
 13.3  Depuracion de datos
----------------------------------------------------------------------
 - Revisar todos los archivos JSON en data/:
   * Integridad: no faltan IDs referenciados
   * Consistencia: tipos correctos, rangos validos
   * Duenos: no hay registros huerfanos
   * Referencias cruzadas: scenario.json references provincias que existen
   * Eventos: condiciones que referencian IDs de formaciones/paises que
     existen

----------------------------------------------------------------------
 13.4  Depuracion de rendimiento
----------------------------------------------------------------------
 - Identificar:
   * Loops que se ejecutan cada frame sin necesidad
   * Funciones _process() pesadas en nodos que deberian usar seniales
   * Diccionarios/arrays que crecen sin limite
   * Instanciacion repetitiva de nodos (deberia ser object pool)
   * Parsing de JSON en cada _ready() (deberia ser preload)
   * Conexiones de senial duplicadas

----------------------------------------------------------------------
 13.5  Depuracion de flujo del juego
----------------------------------------------------------------------
 - Seguir el flujo completo: StartMenu → NationSelect → TestScenario
   * Verificar que no hay pantallas en negro/colgadas entre transiciones
   * Verificar que los autoloads se inicializan en orden correcto
   * Verificar que el estado del juego es consistente en cada paso
   * Verificar que guardar y cargar partida funciona sin perdida de datos
 - Seguir ciclos completos de juego: dia → mes → ano
   * Verificar que los eventos se disparan en el momento correcto
   * Verificar que los modificadores expiran correctamente
   * Verificar que la economia produce/consume correctamente

 FASE 14 — AUDITORIA TOTAL
================================================================================
 Objetivo: Auditoria completa dividida en 3 partes independientes. Cada una
 produce un informe escrito en docs/auditoria/.

----------------------------------------------------------------------
 14.1  Auditoria archivo por archivo, linea a linea
----------------------------------------------------------------------
 - Revisar CADA archivo del proyecto (scripts, escenas, datos, assets)

   * ?Sigue las convenciones del proyecto? (estilo, nombres, estructura)

   * ?Las constantes estan en el lugar correcto?

----------------------------------------------------------------------
